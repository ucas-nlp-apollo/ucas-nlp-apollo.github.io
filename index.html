Few-shot Text Classification System Construction and Validation

Few-shot text classification (FTC) enables training accurate text classification models with limited number of labeled text. We establish a FTC system based on Domain-Tuning to learn the data distribution for this downstream task, achieving superior performance than the state-of-the-art. Let's explore the FTC system built by group Apollo!

Problem Background

Innovation is an inexhaustible driving force for national development and national rejuvenation. In recent years, with the improvement of policy support, national industrialization level and national education level, the number of patent applications in my country has grown rapidly, and the demand for patent search, novelty search and management has also continued to increase. In order to meet the above requirements and improve the quality of patent services, it is usually necessary to establish a multi-dimensional patent classification system. Common classification systems include the International Patent Classification (IPC), the Combined Patent Classification (CPC), the European Patent Classification (ECLA), etc., but these classification systems are relatively complex and highly specialized, and it is difficult for non-IP personnel to use them. Wisdom Buds, as the world's leading intellectual property SaaS platform, has developed a new patent classification system based on users' search habits and other factors, so as to improve the user experience.

Compared State-of-the-art

The competition party disclosed 958 pieces of patent data, including the patentee, patent title, patent abstract and classification labels, among which the classification labels have been desensitized, and there are 36 categories in total. The contestants are required to design a set of algorithms to complete the classification task of the test data. The amount of training data published in this competition is relatively small, which belongs to the classification task based on small sample training data. Small-sample classification tasks have been a hot research topic in recent years. The academic community has proposed solutions such as remote supervision, data enhancement, pre-training models, and PET paradigms. It is hoped that the contestants will give full play to their creativity and apply the research results of the academic circles to this competition topic.

Model Implementation

In our implementation of PLMs, we choose the version "bert-base-chinese", "chinese-roberta-wwm-ext",based on Transformers for BERT, RoBERTa model, respectively. Due to hardware resource limitations, we only retrain the BERT(base) model with masking task, and use it on Domain-Tuning. For Fine-Tuning method, we select the larger model to classify. Specifically, we set batch size to be 12 instances for Fine-Tuning, 16 instances for Domain-Tuning. Dropout rate is set 0.5 for the representations of all models. All models are implemented on Pytorch (version 1.11.0), and model parameters are randomly initialized. For the models with Fine-Tuing and Domain-Tuning, AdamW is used as optimizer, and we use 1e-5 as learning rate.

Data Augmentation

Due to the inadequate of training data, we adopt "Back translation" to increase the amount of our data, in order to rise the effectiveness. We used the machine translation API provided by Tencent company. The datasets are originally in Chinese, after translating every single row of data into five different kind of languages(which are English, Japanese, French, Russian, Malay) originated from different language families, we translate them back into Chinese. This procedure ensures the semanteme of the results are identical with primary data, and generates a slight discrepancy in the expression in the meanwhile. Semantic Segmentation

We assert that it is relatively difficult to classify the data if removing the last sentence of the abstract. Therefore, we divide the abstract of each piece of data into three parts: statement, program and effect. By disassembling this piece of data, we can put a greater weight on the effect part, making it stand out in a large piece of text, thereby improving the effectiveness of the information.

Experimental Results

Taking early fine-tuned Bert as baseline, model performance has been greatly improved with domain-tuning. What's more, by making full use of a small amount of labeled data and a large amount of unlabeled test data, the parameter space of BERT\_D is more suitable for this downstream task. No need to generate more data with the same distribution as the annotated data, BERT_D has best performance without data enhancement, which is useless for BERT_D. Besides, early explorations utilizing knowledge distillation combined with self-training outperforms fine-tuned Bert.
